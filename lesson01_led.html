시작 전 — TIMING은 왜 필요한가

모든 예제 코드에 이 두 줄이 들어 있어요.

setup 
TIMING = (280, 515, 515, 745)
led = NeoPixel(Pin(16), 1, timing=TIMING)

왜 필요한가?

네오픽셀 LED는 데이터선이 딱 한 개예요. 클럭선도 없어요. 그럼 0과 1을 어떻게 구분할까요?

답은 "HIGH 신호가 지속되는 시간의 길이"입니다.

  • 짧게 HIGH → 길게 LOW = 0
  • 길게 HIGH → 짧게 LOW = 1

이 "짧게/길게"의 정확한 길이가 칩마다 조금씩 달라요. 단위는 나노초(ns)이고, timing=(T0H, T0L, T1H, T1L) 네 개 값으로 정합니다.

structure 
TIMING = (280, 515, 515, 745)
#          T0H  T0L  T1H  T1L
#           │    │    │    └─ "1" 신호의 LOW 지속 시간 (ns)
#           │    │    └─ "1" 신호의 HIGH 지속 시간 (ns)
#           │    └─ "0" 신호의 LOW 지속 시간 (ns)
#           └─ "0" 신호의 HIGH 지속 시간 (ns)

MicroPython의 NeoPixel 기본값은 원조 WS2812용이라, 우리가 쓰는 WS2813 Mini에 그대로 적용하면 신호를 0인지 1인지 잘못 읽어요. 그래서 색이 깨지거나 LED가 안 켜져요.

(280, 515, 515, 745)는 WS2813 Mini 데이터시트 스펙 안에 들어가는 안전한 값입니다.

💡 비유

이 값은 "이 LED 부품에 맞는 모스 부호 속도"라고 생각하면 돼요. 속도가 안 맞으면 메시지가 전달이 안 되는 거죠.

⚠️ 디버깅 1순위

LED가 이상하게 깜빡이거나, 요청한 색과 다른 색이 나오거나, 아예 안 켜진다면 TIMING 인자를 빠뜨렸는지부터 확인하세요.

python 
# ❌ 이렇게 쓰면 WS2813 Mini는 제대로 안 켜짐
led = NeoPixel(Pin(16), 1)

# ✅ 이렇게 써야 함
led = NeoPixel(Pin(16), 1, timing=TIMING)

그라데이션 — 직접 RGB + 보간

학습 목표

  • "부드러움 = 단계 수"라는 감각
  • 두 색 사이를 잇는 보간(interpolation) 개념
  • 이 단계가 끝날 무렵 "이걸 일일이 짜는 건 너무 길다"는 답답함을 느끼는 것이 핵심

코드

빨강(255,0,0)에서 주황(255,128,0)으로 한 번에 점프하지 말고, 중간 값을 거쳐 가자는 발상이에요.

python · 03_gradient.py 
# ════════════════════════════════════════
#  3단계: 그라데이션 - 색 사이를 부드럽게 잇기
#  - 두 색 사이를 N단계로 나눠서 보여주면 자연스럽다!
# ════════════════════════════════════════

from machine import Pin
from neopixel import NeoPixel
import time

TIMING = (280, 515, 515, 745)
led = NeoPixel(Pin(16), 1, timing=TIMING)

# 빨강 → 주황으로 부드럽게 변화시켜 보자
# 빨강: (255, 0, 0)
# 주황: (255, 128, 0)
# G값만 0에서 128로 천천히 늘리면 됨!

STEPS = 50   # 50단계로 나누기

start = (255, 0,   0)   # 빨강
end   = (255, 128, 0)   # 주황

for i in range(STEPS + 1):
    # 시작색과 끝색을 i:STEPS-i 비율로 섞기
    r = start[0] + (end[0] - start[0]) * i // STEPS
    g = start[1] + (end[1] - start[1]) * i // STEPS
    b = start[2] + (end[2] - start[2]) * i // STEPS

    led[0] = (r, g, b)
    led.write()
    time.sleep(0.03)

# 잠깐 멈추기
time.sleep(1)

# 이제 주황 → 노랑
start = (255, 128, 0)
end   = (255, 255, 0)

for i in range(STEPS + 1):
    r = start[0] + (end[0] - start[0]) * i // STEPS
    g = start[1] + (end[1] - start[1]) * i // STEPS
    b = start[2] + (end[2] - start[2]) * i // STEPS

    led[0] = (r, g, b)
    led.write()
    time.sleep(0.03)

led[0] = (0, 0, 0)
led.write()
👉 관찰 미션
  • STEPS = 50을 STEPS = 5로 바꿔보세요. 다시 끊겨 보이죠?
  • STEPS = 200으로 늘리면? 더 부드러워지지만 시간이 오래 걸려요.
  • "부드러움 = 단계 수"라는 감각을 몸으로 익히기
🤔 멈춰서 생각해보기

빨→주만 위 코드로 했어요. 빨주노초파남보 7색을 전부 그라데이션으로 연결하려면 위 for 반복문을 6번이나 복사해야 해요. 너무 귀찮죠? 게다가 코드도 지저분해져요.

TIMING = (280, 515, 515, 745) led = NeoPixel(Pin(16), 1, timing=TIMING) 지금까지 빨주노초파남보 7색 사이를 일일이 for 반복문으로 그라데이션 연결했는데, 너무 길고 반복이 많아. 다음 조건으로 깔끔하게 만들어줘: 1. 빨강 → 주황 → 노랑 → 초록 → 파랑 → 남색 → 보라 → 다시 빨강 순서로 부드럽게 그라데이션이 이어져야 해 2. 무지개 한 바퀴 도는 데 약 10초 걸리게 3. 끊김 없이 3바퀴 돈 다음 자동으로 꺼지기 4. 밝기는 너무 눈부시지 않게 40% 정도로 낮춰줘 (그냥 RGB 값에 0.4 곱하면 돼) 5. 코드 맨 위에 내가 바꿔볼 만한 값을 변수로 모아줘: - 한 바퀴 도는 시간 - 총 몇 바퀴 돌지 - 밝기 (0.0 ~ 1.0) 6. 한글 주석 친절하게 힌트로, HSV 색 공간을 쓰면 무지개를 만들기 쉽다고 들었어. HSV의 H값(0~360)을 천천히 바꾸기만 하면 자동으로 빨주노초파남보가 나온다고 하던데, 그 방법을 써도 좋아.

✅ 작동하는 모범 답안

04_rainbow_loop.py Python · ~50줄 
# ════════════════════════════════════════
#  4단계: 무지개 자동 순환 (HSV 활용)
# ════════════════════════════════════════

from machine import Pin
from neopixel import NeoPixel
import time

# ⚠️ WS2813 Mini 전용 타이밍
TIMING = (280, 515, 515, 745)
led = NeoPixel(Pin(16), 1, timing=TIMING)

# ─── 내가 바꿔볼 값 ───────────────────
SECONDS_PER_LAP = 10    # 무지개 한 바퀴 도는 시간 (초)
TOTAL_LAPS      = 3     # 총 몇 바퀴 돌지
BRIGHTNESS      = 0.4   # 밝기 (0.0 ~ 1.0)
FRAME_DELAY     = 0.02  # 프레임 간격 (작을수록 부드러움)
# ────────────────────────────────────────


def hsv_to_rgb(h, s, v):
    """HSV → RGB 변환
    h: 색상 (0~360), s: 채도 (0~1), v: 명도 (0~1)
    H값만 천천히 0→360으로 바꾸면 빨주노초파남보가 자동으로 나옴!
    """
    h = h % 360
    c = v * s
    x = c * (1 - abs((h / 60) % 2 - 1))
    m = v - c
    if   h <  60: r, g, b = c, x, 0
    elif h < 120: r, g, b = x, c, 0
    elif h < 180: r, g, b = 0, c, x
    elif h < 240: r, g, b = 0, x, c
    elif h < 300: r, g, b = x, 0, c
    else:         r, g, b = c, 0, x
    return (int((r + m) * 255), int((g + m) * 255), int((b + m) * 255))


# 한 바퀴 = 360도. 프레임 하나당 몇 도 움직일지 계산
frames_per_lap = int(SECONDS_PER_LAP / FRAME_DELAY)
hue_step = 360 / frames_per_lap

print("🌈 무지개 순환 시작!")

hue = 0
for lap in range(TOTAL_LAPS):
    print(f"  {lap + 1}/{TOTAL_LAPS} 바퀴")
    for _ in range(frames_per_lap):
        led[0] = hsv_to_rgb(hue, 1, BRIGHTNESS)
        led.write()
        hue += hue_step
        time.sleep(FRAME_DELAY)

led[0] = (0, 0, 0)
led.write()
print("✅ 끝!")

05_emotion_led.py Python · ~120줄 
# ════════════════════════════════════════
#  5단계: 감정 표현 LED (3가지 모드)
# ════════════════════════════════════════

from machine import Pin
from neopixel import NeoPixel
import time
import math

# ⚠️ WS2813 Mini 전용 타이밍
TIMING = (280, 515, 515, 745)
led = NeoPixel(Pin(16), 1, timing=TIMING)


def dim(color, factor):
    """색의 밝기를 factor 배로 줄이기 (0.0~1.0)"""
    return (int(color[0] * factor),
            int(color[1] * factor),
            int(color[2] * factor))


# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
#  💓 모드 1: 심장박동
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
def heartbeat():
    # ─── 바꿔볼 값 ───
    COLOR    = (255, 0, 60)   # 빨강
    BEATS    = 5              # 몇 번 반복할지
    BRIGHT   = 0.4            # 최대 밝기
    # ─────────────────

    print("💓 심장박동")
    for _ in range(BEATS):
        # 쿵! (1차 박동, 작게)
        for f in [0.2, 0.6, 1.0, 0.6, 0.2]:
            led[0] = dim(COLOR, f * BRIGHT)
            led.write()
            time.sleep(0.06)
        time.sleep(0.05)

        # 쿵! (2차 박동, 더 크게)
        for f in [0.3, 0.8, 1.0, 0.8, 0.3, 0.1]:
            led[0] = dim(COLOR, f * BRIGHT)
            led.write()
            time.sleep(0.06)

        # 쉬는 시간
        led[0] = (0, 0, 0)
        led.write()
        time.sleep(0.6)


# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
#  🚨 모드 2: 사이렌
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
def siren():
    # ─── 바꿔볼 값 ───
    RED      = (255, 0, 0)
    BLUE     = (0, 50, 255)
    CYCLES   = 10
    SPEED    = 0.12   # 한 번 켜지는 시간
    BRIGHT   = 0.4
    # ─────────────────

    print("🚨 사이렌")
    for _ in range(CYCLES):
        led[0] = dim(RED, BRIGHT)
        led.write()
        time.sleep(SPEED)

        led[0] = (0, 0, 0)
        led.write()
        time.sleep(0.05)

        led[0] = dim(BLUE, BRIGHT)
        led.write()
        time.sleep(SPEED)

        led[0] = (0, 0, 0)
        led.write()
        time.sleep(0.05)


# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
#  🌊 모드 3: 평온한 호흡
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
def calm_breath():
    # ─── 바꿔볼 값 ───
    COLOR    = (100, 200, 255)   # 하늘색
    BREATHS  = 4
    PERIOD   = 4.0    # 한 호흡(들숨+날숨) 시간 (초)
    BRIGHT   = 0.4
    # ─────────────────

    print("🌊 평온한 호흡")
    FRAME_DELAY = 0.03
    frames = int(PERIOD / FRAME_DELAY)

    for _ in range(BREATHS):
        for i in range(frames):
            # 사인파로 부드럽게 들숨-날숨
            phase = (i / frames) * 2 * math.pi
            level = (math.sin(phase - math.pi / 2) + 1) / 2  # 0~1
            led[0] = dim(COLOR, level * BRIGHT)
            led.write()
            time.sleep(FRAME_DELAY)


# ╔══════════════════════════════════════════╗
# ║          메인 - 3개 모드 순서대로        ║
# ╚══════════════════════════════════════════╝

heartbeat()
time.sleep(1)

siren()
time.sleep(1)

calm_breath()
time.sleep(1)

led[0] = (0, 0, 0)
led.write()
print("✅ 끝!")
